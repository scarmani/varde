# MCTS Search V5 — Learnings

## Carried evidence

- V4 selected `none-qualified`; scaling a failed architecture is prohibited.
- Separate modules are not independent when they duplicate the same semantic
  recursion. V5 proof traces must record actor ownership and use a declarative
  oracle that shares no obligation implementation with the solver.
- Immediate fence completion and one-reply fence durability are different
  propositions and require separate labels.
- Repeated interior solver scans were avoidable; root-only guidance is the V5
  architecture.
- V4 registered roots did not isolate widening's intended wide-root failure.
  V5 freezes mandatory wide roots and an explicit instrument-validity gate.
- Settling V1 called ordinary placements progress and was slower. V2 removes
  generic score/control changes and must earn both efficiency and admission.

## Update rule

Add only evidence-backed, reusable findings. Keep transient debugging in the
execution log. Failed hypotheses and invalid instruments are completed results,
not invitations to tune a frozen gate.
